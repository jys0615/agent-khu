### **SLIDE 5: MCP 프로토콜 상세**
```
제목: "MCP (Model Context Protocol) = AI의 표준화된 도구 사용법"

좌측: MCP의 개념 (3단계)

1️⃣ Initialize
┌──────────────┐
│ Client       │
│ → Server     │ "나를 초기화해"
│ ← Capabilities│ "이게 내 능력이야"
└──────────────┘

2️⃣ Discover Tools
┌──────────────┐
│ Client       │
│ → tools/list │ "뭘 할 수 있어?"
│ ← Tool List  │ "이것들 할 수 있어"
└──────────────┘

3️⃣ Call Tool
┌──────────────────────┐
│ Client               │
│ → tool/call          │ "이거 해줄래?"
│   {tool: get_notice} │
│ ← result             │ "결과: ..."
└──────────────────────┘

우측: 장점 (박스형)
┌────────────────────┐
│ 🟢 표준화         │
│   OpenAI, Google  │
│   모두 지원       │
│                  │
│ 🟢 안전성        │
│   타입 검증      │
│   권한 관리      │
│                  │
│ 🟢 확장성       │
│   새 서버 추가   │
│   자동 인식     │
│                  │
│ 🟢 효율성       │
│   Lazy Start    │
│   필요시만 실행 │
└────────────────────┘

하단: JSON-RPC 2.0 프로토콜 예시
┌──────────────────────────────┐
│ 요청:                        │
│ {"jsonrpc": "2.0",          │
│  "id": 1,                   │
│  "method": "tools/call",    │
│  "params": {                │
│    "name": "get_notice",    │
│    "arguments": {...}       │
│  }                          │
│ }                           │
│                            │
│ 응답:                       │
│ {"jsonrpc": "2.0",         │
│  "id": 1,                  │
│  "result": {               │
│    "content": [...]        │
│  }                         │
│ }                          │
└──────────────────────────────┘

### **SLIDE 6: Agent Loop (의사결정 과정)**
```
제목: "Claude의 자율적 의사결정: Agent Loop"

중앙: 시나리오 예시
┌────────────────────────────────┐
│ 사용자 질문:                   │
│ "공지 3개 알려줄래?"          │
└────────────────────────────────┘

반복 과정 (좌→우 흐름):

Loop 1️⃣ [Claude 분석]
┌──────────────────────┐
│ Claude 생각:         │
│ "공지를 찾으려면     │
│  학과 정보가 필요해" │
│                     │
│ 결정: Tool 호출     │
│ → get_latest_notices│
└──────────────────────┘

Loop 2️⃣ [MCP 호출]
┌──────────────────────┐
│ Backend에서:        │
│ notice-mcp 실행     │
│ ↓                  │
│ 웹 크롤링 수행     │
│ (공지사항 수집)     │
│                     │
│ 결과: 2개 공지     │
└──────────────────────┘

Loop 3️⃣ [결과 분석]
┌──────────────────────┐
│ Claude 분석:        │
│ "3개 요청했는데     │
│  2개만 찾았네"     │
│                     │
│ 결정: 추가 검색?   │
│ 아니면 사용자에게   │
│ 제시할까?          │
└──────────────────────┘

Loop 4️⃣ [최종 응답]
┌──────────────────────┐
│ Claude 응답:        │
│ "공지 2개 찾았어:   │
│  1. 국제교육원...   │
│  2. 졸업논문..."    │
│                     │
│ → 사용자에게 표시  │
└──────────────────────┘

우측: Agent Loop 특징
┌────────────────┐
│ 최대 5 iterations│
│                 │
│ ✅ 복잡한 문제  │
│    자율 해결    │
│                 │
│ ✅ 부분 실패    │
│    대체책 시도  │
│                 │
│ ✅ 맥락 누적    │
│    이전 결과    │
│    기반 다음    │
│    단계 결정    │
└────────────────┘

화살표: 원형 (반복의 의미) + 진행도 표시
색상: 각 loop마다 다른 색상 (#FFE57F, #81C784, #64B5F6, #E57373)
```

"안녕하세요, 정윤서입니다.

저는 오늘 '캠퍼스 정보 산재 문제'를 AI로 해결한 
프로젝트 'Agent KHU'를 소개하겠습니다.

여러분이 학교 다닐 때 이런 경험 있으신가요?
- 강의실 위치를 찾으려고 학사시스템, 캠퍼스맵, 카톡 등을 뒤지는 경험
- 수강신청은 학사시스템이 너무 느려서 기다려야 하는 경험
- 학식 메뉴는 따로 앱을 다시 켜야 하는 경험

문제는 이런 정보들이 모두 다른 곳에 산재되어 있다는 것입니다.

우리의 솔루션은 간단합니다.
1개의 자연어 인터페이스로 7개의 정보 서비스를 통합하는 것입니다.

'공지 3개 알려줄래?' 라고 물으면
AI가 자동으로 공지를 크롤링하고, 분석해서,
맞춤형 답변을 제공합니다.

혁신점은:
- 통합: 산재된 정보를 1곳으로
- 자동화: 실시간 크롤링
- AI 맥락 이해: 학번, 학과 기반 개인화
- 확장성: 7개 서버에서 N개로 확장 가능
"
```

### Slide 4-7 (기술, 4분)
```
"이제 기술 구조를 설명하겠습니다.

(Slide 4 보여주기)
이것은 Agent KHU의 전체 시스템입니다.
맨 위는 사용자가 사용하는 React 기반 프론트엔드.
중간은 FastAPI 백엔드로, 여기서 Claude AI가 핵심 역할을 합니다.
아래는 7개의 MCP 서버들 - 공지, 교과과정, 학식 등등.
맨 아래는 데이터 소스 - 데이터베이스와 웹 크롤링입니다.

(Slide 5 보여주기)
이 프로젝트의 핵심 기술은 MCP(Model Context Protocol)입니다.
이는 Anthropic, OpenAI, Google이 공동으로 정의한 표준 프로토콜입니다.

간단히 설명하면:
AI가 도구를 사용하는 표준화된 방법입니다.

초기화 → 도구 목록 조회 → 도구 호출
이렇게 3단계로 진행됩니다.

장점은:
1. 표준화: 어느 LLM과도 호환
2. 안전성: 타입 검증
3. 확장성: 새 서버 추가하면 자동으로 AI가 인식
4. 효율성: Lazy Start로 필요한 서버만 실행

(Slide 6 보여주기)
AI는 자율적으로 다음을 반복합니다:
1단계: Claude가 분석 - '이 질문 답변하려면 공지 정보가 필요하네'
2단계: Tool 호출 - get_latest_notices 호출
3단계: 결과 수신 - MCP를 통해 공지사항 데이터 수신
4단계: 분석 및 응답 - 사용자에게 최종 답변

최대 5회 반복으로 복잡한 질문도 해결합니다.
이걸 Agent Loop이라고 부릅니다.
"
