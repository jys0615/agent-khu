## 📋 종합 현황 분석

### 1. 프로젝트 규모 및 기술 스택

**시스템 구조**:
- **Frontend**: React + TypeScript + Vite + TailwindCSS (모던 UI)
- **Backend**: FastAPI (비동기 웹 서버)
- **AI Agent**: Claude Sonnet 4 (LLM 기반 의사결정)
- **Integration Protocol**: MCP (Model Context Protocol 2024-11-05)
- **Database**: PostgreSQL 15 (구조화 데이터)
- **Cache**: Redis 7 (성능 최적화)
- **Search**: Elasticsearch 8.11 (로깅/분석)
- **Infrastructure**: Docker Compose (컨테이너화)

**마이크로서비스** (7개 MCP 서버):
1. curriculum-mcp - 교과과정 (자동 갱신)
2. notice-mcp - 공지사항 (실시간 크롤링)
3. course-mcp - 수강신청 (자동화)
4. library-mcp - 도서관 (실시간 예약)
5. meal-mcp - 학식 (메뉴 검색)
6. shuttle-mcp - 셔틀버스 (도착 정보)
7. classroom-mcp - 강의실 (위치 검색)

#### Slide 5: MCP 프로토콜의 장점
```
MCP (Model Context Protocol) = LLM이 도구를 사용하는 표준화된 방법

장점:
1. 표준화: OpenAI, Anthropic 공통 지원
2. 안전성: 타입 검증, 권한 관리
3. 확장성: 새 서버 추가 = 새 tool 자동 인식
4. Lazy Start: 필요한 서버만 동작 → 메모리 절약

기술 세부사항:
- JSON-RPC 2.0 over stdio
- Initialize → tools/list → tool/call
- 비동기 처리 (asyncio)
```

#### Slide 6: Agent Loop (Claude의 자율적 의사결정)
```
사용자: "공지 3개 알려줄래?"

Loop 반복:
1️⃣ Claude: "학생 정보 필요" → get_latest_notices 호출
2️⃣ Backend: notice-mcp 호출 → 크롤링 실행
3️⃣ Claude: 결과 분석 → "3개는 없네, 2개 찾음"
4️⃣ Claude: "이거 할까?" → 사용자 보여줌

최대 5 iterations = 자율적 문제해결
```

#### Slide 7: 데이터 흐름 (캐싱 + 최적화)
```
요청 → Redis 확인
       ✓ Hit (18% 개선) → 즉시 반환
       ✗ Miss → MCP 호출 → DB/Crawling → 저장

캐시 성능:
- 공지: 15.94s → 13.10s (18% ↓)
- 학식: 8.44s → 8.09s (4% ↓)

구조: 쿼리별 TTL (공지 2h, 학식 12h)
```
