### 문제 정의
```
학생들이 캠퍼스 정보를 얻기 위해 7곳 이상의 시스템을 사용해야 함
- 강의실: 학사시스템 + 캠퍼스맵
- 수강신청: 학사시스템 (느림)

- 셔틀버스: 별도 앱

→ **불편함, 시간 낭비, 정보 오류율 높음**
```


## 🔧 기술 아키텍처 (차별화)

### 핵심 기술
```
1. MCP (Model Context Protocol) ← 표준화된 AI 도구 인터페이스
2. Agent Loop (Claude) ← 자율적 의사결정 (최대 5회 반복)
3. Lazy Start MCP Servers ← 효율성 (메모리 절약)
4. 마이크로서비스 ← 확장성 (7→N 서버)
```

### 기술 스택
| 계층 | 기술 | 특징 |
|------|------|------|
| **Frontend** | React + TypeScript + TailwindCSS | 모던 UI |
| **Backend** | FastAPI | 비동기 처리 |
| **AI** | Claude Sonnet 4 | LLM 기반 |
| **Protocol** | MCP (JSON-RPC 2.0) | 표준 프로토콜 |
| **Database** | PostgreSQL 15 | 구조화 데이터 |
| **Cache** | Redis 7 | 성능 최적화 |
| **Logging** | Elasticsearch 8.11 | 모니터링 |
| **Container** | Docker Compose | 배포 자동화 |

### MCP 서버 (7개)
1. **curriculum-mcp** - 교과과정 (자동 갱신)
2. **notice-mcp** - 공지사항 (실시간 크롤링)
3. **course-mcp** - 수강신청 (Playwright 자동화)
4. **library-mcp** - 도서관 (실시간 예약)
5. **meal-mcp** - 학식 (메뉴 검색)
6. **shuttle-mcp** - 셔틀버스 (도착 정보)
7. **classroom-mcp** - 강의실 (위치 검색)
